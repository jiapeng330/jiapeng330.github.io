配置服务器并且将网站挂到服务器上全过程【精】

刚刚学习服务器，我使用了WDCP管理系统，这个框架对于刚学习服务器的菜鸟还是非常好用的，网上的参考资料也比较多。

机子是阿里云ECS，配置详情如下：

一、安装WDCP

安装方法详情见参考链接：https://www.wdlinux.cn/wdcp/install.html

源码安装(ssh登录服务器,执行如下操作即可,需root用户身份安装)
wget http://dl.wdlinux.cn/lanmp_laster.tar.gz
tar zxvf lanmp_laster.tar.gz
sh lanmp.sh
默认安装N+A的引擎组合

安装完后,默认的后台管理地址如下
http://ip:8080
用户名:admin 默认密码:wdlinux.cn
mysql默认的管理用户名:root 默认密码:wdlinux.cn

相关说明
所有软件安装目录/www/wdlinux
站点配置文件
/www/wdlinux/nginx/conf/vhost
/www/wdlinux/apache/conf/vhost
数据库配置文件/www/wdlinux/etc/my.cnf
数据库数据文件目录 /www/wdlinux/mysql/var 或 /www/wdlinux/mysql/data

安装完成：

二、连接服务器

这里有3种连接方法：
1.在命令行里通过ssh root@你的服务器ip地址 -p 8080

2.3.通过Workbench和VNC连接

三、安装Tomcat

在课程的学习中，我们通过Tomcat将网站挂在服务器上。

Tomcat源：http://mirrors.hust.edu.cn/apache/tomcat/tomcat-8/v8.5.51/bin/apache-tomcat-8.5.51.tar.gz

找到合适的目录后，使用wget来下载Tomcat8：

wget http://mirrors.hust.edu.cn/apache/tomcat/tomcat-8/v8.5.51/bin/apache-tomcat-8.5.51.tar.gz

然后解压

tar -zxvf apache-tomcat-8.5.51.tar.gz

可以使用 mv命令来重命名和移动

Tomcat8 安装完成！

但是！！！Tomcat的默认端口是8080 与WDCP的管理默认端口8080相同，必须将Tomcat或者WDCP的默认端口修改成空闲的端口。否则8080端口就被Tomcat霸占啦！准没好事情。

下面是修改Tomcat和WDCP的默认端口方法

Tomcat端口修改：

通过vim打开server.xml修改：
vim apache-tomcat-8.5.51/conf/server.xml

这里我将8080端口改成了9330

按 ESC :wq 保存退出

重启Tomcat：

cd apache-tomcat-8.5.51/bin

./shutdown.sh

./startup.sh

这里发现并不能访问118.31.123.66:9330

这里困扰了我很久，一直以为是CentOS7的防火墙firewall搞的鬼，最后才发现，因为这是阿里云ECS，需要到云服务控制台设定安全性规则打开对应的端口才可以访问。
创建新的安全性规则

建立好后就可以成功访问了！

WDCP端口修改：

vim www/wdlinux/wdcp/conf/port.conf

修改文件中的端口号8080为新的端口号

按 ESC :wq 保存后重启wdcp：

service wdcp restart

WDCP端口修改完成！同样要到设定上面的安全性规则。

四、挂载静态网站

这里呢我用FileZilla连接服务器，主机输入sftp://你的ip地址:22，用户名root加密码。

将静态的Project这里是Web Application中的文件上传到/usr/Tomcat/apache-tomcat-8.5.51/webapps/bbs下

重启Tomcat便可以输入IP:8080/bbs访问index首页啦。

五、挂载动态网站

首先呢 Build Artifacts 创建WAR包

导出写好的Web Application后会生成在工程名的\out\artifacts目录下

同挂载静态网站一样，将该目录下的文件上传到/usr/Tomcat/apache-tomcat-8.5.51/webapps/bbs下

重启Tomcat后能访问网页，但是没有数据。

接下来就是将MySQL中的数据库的表导入到服务器创建的数据库中

访问WD Control Panel，进入管理界面创建同名数据库（我这里是“bbs”），然后进入phpMyAdmin，输入连接MySQL的账号密码，在刚刚建好的数据库中进行导入表的操作。

到这里呢，数据库已经导入成功了。但是通过service mysqld restart重启后还是不能在网页读取数据库中的数据。

那是因为Tomcat和MySQL之间还缺少了一个JDBC驱动

安装JDBC

安装JDBC呢简直不要太简单，一句话搞定：
sudo yum install mysql-connector-java

最后一步，修改MySQL中的/etc/my.cnf文件（通过whereis mysql查看安装位置），在[mysqld]下加入一行：lower_case_table_names=1

因为在Linux系统下的MySQL和Windows下的不太一样，是区分大小写的（就是因为这个困扰了我好久哈哈），这里呢我们这样处理就把表名的大小写忽略了。

大功告成！jiapeng330书。
